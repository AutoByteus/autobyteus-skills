# Mainline coherence gate (prevent “off-topic but factual” drift)

Use this after logic/evidence checks and before finalizing `article.md`.

## Goal

Ensure the article is not only factual, but also *on the exact user-intended main line*.

## Define the main line first

Write one sentence:
- **Mainline statement:** “For [audience], this article proves [core claim] by showing [3–6 moves], and does **not** focus on [out-of-scope items].”

If you cannot write this clearly, do not finalize the draft.

## Section-by-section relevance test

For each major section/move, ask:
1) If this section is removed, does the thesis become weaker or unclear?
2) Does this section directly support one roadmap move?
3) Is its key evidence used later in reasoning/application?

If answers are mostly “no,” the section is off-mainline and must be cut, merged, or reframed.

## Off-mainline red flags

- A vivid historical/context point appears once and never contributes to downstream reasoning.
- A data point is true but not decision-relevant for the user’s question.
- A paragraph introduces a new mini-topic that the thesis does not require.
- Background takes more space than the argument it is supposed to support.
- The reader could ask “So what?” and the section has no clear answer.

## Required fix actions

When a red flag appears, choose one:
- **Cut:** remove the section/line entirely.
- **Compress:** keep one line and move on.
- **Reframe:** rewrite explicitly as support for a specific move.
- **Relocate:** move to objection/limitations if it is context, not core argument.

## Pass criteria

All must be true:
- Every section maps to a roadmap move.
- No load-bearing sentence is “interesting but optional.”
- No slide headline would need a section that fails relevance test.
- Opening context does not dominate the body argument.
