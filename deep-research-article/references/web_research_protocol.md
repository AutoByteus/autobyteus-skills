# Web research protocol (required by default)

Use this protocol unless the user explicitly requests no-web research.

## Phase A — Search map

Run in this order:
1. Broad framing queries (definitions, baseline statistics, core debates)
2. Focused mechanism queries (causal explanations, studies)
3. Counter-position queries ("critique", "limitations", "replication", "null results")
4. Latest updates (if topic is time-sensitive)

Keep a query log in `source_dossier.md`.

## Phase B — Source selection

Prioritize:
- Primary sources: original papers, official docs, datasets, regulations, first-party reports
- Then high-quality secondary synthesis

Avoid relying only on:
- blog opinion pieces
- low-transparency summaries
- single-source claims

## Phase C — Deep reading

For each selected source, extract:
- exact claim (or data point)
- method/context (who, where, when, sample/task)
- limitation/caveat
- relevance to your thesis

Do not jump to writing until extraction is complete.

## Phase D — Triangulation

For each load-bearing claim:
- verify with at least 2 independent sources, OR
- present as tentative with explicit caveat

Include at least 2 meaningful counterevidence sources in deep mode.

## Phase E — Stop rule before drafting

You may draft `article.md` only after:
- source minimums are met
- evidence extraction table is complete
- claim→evidence ledger exists for major claims
