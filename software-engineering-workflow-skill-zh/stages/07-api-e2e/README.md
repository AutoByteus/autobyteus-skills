# 阶段 7 API/E2E 和可执行验证

## 目的

使用适合系统的可执行验证场景，针对验收标准和相关运行时主干验证已实现的行为，包括 API、UI、原生桌面、CLI、进程/生命周期、集成或分布式流程（视需要而定）。

## 进入此阶段条件

- 阶段 6 已完成
- 源代码编辑、单元测试和集成测试处于足够稳定的状态，可以跨范围内接口和运行时边界验证可执行行为

## 阶段拥有的输出

- `tickets/in-progress/<ticket-name>/api-e2e-testing.md`
- 场景实现和执行结果

## 退出门控

仅在所有可执行的范围内验收标准为 `Passed` 或被用户针对不可行情况明确 `Waived`，且相关的主要、返回/事件和有界局部主干有匹配证据或明确的 `N/A` 理由时，才离开阶段 7。
