# 阶段 8 代码审查

## 目的

在阶段 7 验证通过后运行独立的代码审查门控。

## 进入此阶段条件

- 阶段 7 为 `Pass`
- `workflow-state.md` 已移至阶段 8 并锁定源代码编辑

## 阶段拥有的输出

- `tickets/in-progress/<ticket-name>/code-review.md`
- 最终阶段 8 决策、规范优先级排序审查评分卡，以及任何重入声明

## 退出门控

仅在审查决策为 `Pass`、所有强制审查检查均满足、详细的优先级排序审查评分卡已记录、无评分卡分类低于 `9.0`、且分类 `1` 确认相关审查路径的主干清单清晰度和主干跨度充分性时，才离开阶段 8。

## 本地文件

- `code-review-guide.md`：阶段执行和门控规则
- `code-review-principles.md`：共享的阶段 8 审查理念和硬性审查原则
- `code-review-template.md`：规范的审查产物
