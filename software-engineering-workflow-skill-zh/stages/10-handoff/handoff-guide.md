# 阶段 10 移交指南

## 目的

阶段 10 将工程完成转化为最终交付记录。
它分离：

- 工程完成
- 用户验证
- 工单归档
- 仓库定稿
- 适用时的发布 / 出版 / 部署
- 适用时的定稿后清理

## 输入

- 已完成的实现、阶段 7、阶段 8 和阶段 9 输出
- 当前 `workflow-state.md`

## 规范产物

- 创建或更新 `tickets/in-progress/<ticket-name>/handoff-summary.md` 作为规范的阶段 10 产物。
- 使用 `handoff-summary-template.md` 作为起始结构。
- 工单移至 `tickets/done/<ticket-name>/` 后，此文件应随工单归档移动。

## 移交摘要要求

至少记录：

- 交付范围与计划范围对比
- 验证摘要
- 已更新文档或无影响理由
- 发布说明状态
- 清理状态

## 用户验证暂停

- 移交摘要写入后，保持阶段 10 开放。
- 在用户明确确认完成或验证之前，不要将工单移至 `done`、提交、推送、合并或运行发布/出版/部署工作。

## 工单归档

- 阶段 10 拥有从 `tickets/in-progress/<ticket-name>/` 到 `tickets/done/<ticket-name>/` 的移动。
- 仅在用户明确确认或明确移动指令后执行该移动。
- 在最终提交前移动工单，使提交状态包含归档的工单路径。
- 如果工单稍后重新打开，下一个阶段 0 引导应在任何新更新前将其移回 `tickets/in-progress/<ticket-name>/`。

## 发布说明

当工单产生面向用户的发布或 GitHub Release 正文时：

- 创建 `tickets/in-progress/<ticket-name>/release-notes.md`
- 保持说明简短且面向用户
- 省略内部重构、测试、仅文档变更和低级实现细节
- 工单归档后，将归档的 `tickets/done/<ticket-name>/release-notes.md` 产物传入发布/出版路径（当适用时）

## Git 仓库的定稿顺序

用户明确确认后：

1. 将 `tickets/in-progress/<ticket-name>/` 移至 `tickets/done/<ticket-name>/`。
2. 在工单分支上提交所有范围内变更，包括已移动的工单文件。
3. 推送工单分支。
4. 从远程更新已解析的阶段 0 目标分支。
5. 将工单分支合并到该更新后的目标分支。
6. 推送更新后的目标分支。

除非用户稍后明确覆盖，使用阶段 0 已解析的基础远程和基础分支作为默认定稿目标。

## 发布 / 出版 / 部署（适用时）

- 仓库定稿后，仅在该步骤实际适用时运行项目已记录的发布/出版/部署方法。
- 可能是：
  - 发布脚本
  - 已记录的命令
  - git 标签工作流
  - GitHub Release 创建
  - 其他部署或出版路径
- 如果无此类步骤适用，在移交摘要中记录 `发布/出版/部署不需要` 且不阻塞合并/定稿。
- 如果需要发布说明，在该发布/出版路径运行时传入归档的 `tickets/done/<ticket-name>/release-notes.md` 产物。

## 定稿后清理（适用时）

- 如果阶段 0 为此工单创建或复用了专用工单工作树/分支，清理是阶段 10 完成的一部分而非可选项。
- 仓库定稿和任何适用的发布/出版/部署工作完成后：
  - 如需要，在移除工单工作树前切换到安全的父仓库检出，
  - 移除阶段 0 中记录的专用工单工作树，
  - 运行工作树清理，
  - 当本地工单分支已完全合并到已解析的定稿目标且不再需要时，删除该本地工单分支。
- 除非用户明确指示或已记录的项目策略要求，不要删除远程分支。

## 阻塞项

如果任何移动、提交、推送或合并步骤失败：

- 在 `workflow-state.md` 中记录阻塞项
- 保持阶段 10 开放
- 在阻塞解除前不要标记工作流完成

如果适用的发布/出版/部署步骤失败或未记录：

- 在 `workflow-state.md` 中记录该阻塞项
- 保持阶段 10 开放
- 不要撤销已完成的仓库定稿

如果所需的工作树/分支清理失败：

- 在 `workflow-state.md` 中记录该阻塞项
- 保持阶段 10 开放
- 在清理完成前不要视工单为完全完成

## 退出门控

此阶段仅在收到用户明确验证、所有必需的归档/仓库定稿工作完成、任何适用的发布/出版/部署工作完成或明确记录为不需要、且适用时所需的定稿后清理完成时才算完成。
