# Stage 1 Investigation

## Purpose

Build current-state understanding before requirements are finalized for design.
This stage reduces uncertainty, identifies real boundaries and owners, and decides the workflow depth (`Small` / `Medium` / `Large`).

## Enter This Stage When

- Stage 0 has completed
- later work reveals unclear or cross-cutting issues that require refreshed understanding

## Stage-Owned Outputs

- `tickets/in-progress/<ticket-name>/investigation-notes.md`
- scope triage result with rationale
- current entrypoints, owners, boundaries, and constraints recorded

## Exit Gate

Leave Stage 1 only when `investigation-notes.md` is current, the understanding pass is complete, and scope triage is recorded with enough clarity to refine `requirements.md` to `Design-ready`.

## Local Files

- `investigation-guide.md`: investigation method, minimum outputs, and investigation rules
