# Stage 3 Design

## Purpose

Define the target architecture shape before implementation starts.
This stage produces the design basis that later runtime modeling, implementation, and review stages challenge and refine.

## Enter This Stage When

- `requirements.md` is `Design-ready` or `Refined`
- Stage 1 understanding is current enough to support architecture decisions
- a classified re-entry requires design changes

## Stage-Owned Outputs

- `Small`: design section in `implementation.md`
- `Medium` / `Large`: `tickets/in-progress/<ticket-name>/proposed-design.md`

## Exit Gate

Leave Stage 3 only when the design basis is current for the ticket scope and clear enough to drive Stage 4 future-state runtime call stacks.

## Local Files

- `design-guide.md`: stage execution guidance and design rules
- `proposed-design-template.md`: canonical design artifact structure
