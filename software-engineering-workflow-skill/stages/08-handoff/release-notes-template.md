# Release Notes Template

Use this template for `tickets/in-progress/<ticket-name>/release-notes.md` or `tickets/done/<ticket-name>/release-notes.md` when the ticket leads to a user-facing app release or a GitHub Release body.

Rules:

- user-facing only
- functional changes only
- 3 to 7 bullets total across all sections
- one line per bullet
- do not include refactors, dependency bumps, tests, docs-only changes, or internal implementation detail
- omit empty sections instead of writing filler text
- do not include upgrade steps unless the user explicitly asks for them

Template:

```md
## What's New
- Added ...

## Improvements
- Improved ...

## Fixes
- Fixed ...
```
